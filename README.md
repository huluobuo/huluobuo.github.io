# 个人网站

一个简洁美观的个人网站，包含音乐播放和文件下载功能。

## 功能特性

- 🎵 **音乐播放器** - 在线播放音乐列表中的歌曲
- 📁 **文件下载** - 提供文件下载服务
- 🎨 **现代化UI** - 响应式设计，支持移动端
- 🔍 **搜索功能** - 音乐列表支持搜索

## 项目结构

```
.
├── index.html          # 主页
├── music.html          # 音乐列表页面
├── file.html           # 文件下载页面
├── musiclist.json      # 音乐列表数据
├── filelist.json       # 文件列表数据
├── img/                # 图片资源
│   ├── 背景.png        # 背景图片
│   ├── 头像.png        # 头像
│   └── github_logo.png # GitHub图标
├── musics/             # 音乐文件目录
└── files/              # 文件目录
```

## 使用说明

### 音乐播放

1. 访问 `music.html` 页面
2. 点击任意歌曲开始播放
3. 使用播放器控制按钮进行播放控制
4. 使用搜索框快速查找歌曲

### 文件下载

1. 访问 `file.html` 页面
2. 浏览可用文件列表
3. 点击"下载文件"按钮下载

## 数据格式

### musiclist.json

音乐列表是一个字符串数组，每个元素是音乐文件名（不含扩展名）：

```json
[
    "歌曲名称1",
    "歌曲名称2"
]
```

### filelist.json

文件列表是一个对象数组，每个对象包含：

```json
[
    {
        "name": "文件名",
        "size": "文件大小",
        "path": "文件路径",
        "description": "文件描述"
    }
]
```

## 技术栈

- HTML5
- CSS3 (包含动画和渐变效果)
- JavaScript (原生ES6+)
- 响应式设计

## 浏览器支持

- Chrome (推荐)
- Firefox
- Safari
- Edge

## 注意事项

- 确保音乐文件存放在 `musics/` 目录下，文件名与 `musiclist.json` 中的名称匹配（需添加 `.mp3` 扩展名）
- 确保文件存放在 `files/` 目录下，路径与 `filelist.json` 中的路径匹配
- 图片资源需存放在 `img/` 目录下
